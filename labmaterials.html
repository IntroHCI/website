Lab 1: Source control, HTML, servers

Submit your complete lab at http://hci.st/147lab-complete by the end of the lab.

Slide: Welcome
  • Install Git
  • Get a GitHub account
  • Select or download a text editor (e.g., Sublime Text for Mac and Notepad++ for Windows)
  • Install VirtualBox
  • Install Vagrant
  • Debug (OS X): [appname] can't be opened becuase it is from an unidentified developer. Control-click on the file and choose "Open", and you'll get an additional "Open" option when the window pops up.
Slide: Prepare our configuration files
  • Download our class's development machine configuration files and unzip them to create a directory where you plan to keep your code for the class. For example, ~/Documents/introHCI.
Slide: Run vagrant up in the introHCI directory
  • Debug: SSH connection was refused! Do what it suggests and try vagrant reload, or if that doesn't work, vagrant destroy followed by vagrant up.
  • Debug: Vagrant cannot forward the specified ports on this VM, since they would collide with some other application that is already listening on these ports. The forwarded port to 3000 is already in use on the host machine. You might have another application open that's using port 3000. The command lsof -i | grep LISTEN might tell you which one, and you can kill it. A restart might clear the port too. In the worst case, you can go into the Vagrantfile and change the line config.vm.forward_port 3000, 3000 to config.vm.forward_port 3000, 3001 and load the page at http://localhost:3001 instead.
